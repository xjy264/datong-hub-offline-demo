# 单张图片上传上限调整为 50MB 设计

关联 Issue：#1

## 背景与目标

当前 JPEG、PNG、WebP 图片的前后端单张限制均为 20MB，单次 multipart 请求限制为 100MB。目标是让单张不超过 50MB 的图片稳定上传，同时保持每批最多 20 张，并避免多张大图组成的请求超过 100MB。

## 方案选择

采用按“张数与总字节数”双重拆批：

- 单张上限调整为 50MB；
- 每批仍最多 20 张；
- 前端每批文件总大小控制在 90MB 内，为 multipart 边界和字段开销预留空间；
- 后端继续保留 100MB 单次请求上限，避免扩大服务内存峰值；
- 同步页面提示、Docker 环境变量和 README。

未选择仅替换 20MB 常量，因为两张接近 50MB 的图片会因 multipart 开销超过请求限制。未选择将请求上限扩大至 1GB，因为这会增加低配置部署的内存风险。

## 数据流与错误处理

用户选择图片后，前端先拒绝超过 50MB 的文件，再按 20 张和 90MB 两个边界顺序拆分并逐批上传。后端继续校验文件类型、每批数量和单张大小；超过上限时统一返回“单张图片不能超过50MB”。

## 验收标准

- 50MB 以内图片通过前后端单张校验；
- 超过 50MB 的图片在发起请求前得到明确提示；
- 多张大图按总大小拆批，任一批次最多 20 张且不超过 90MB；
- Docker 与默认 Spring 配置使用 50MB 单张上限和 100MB 请求上限；
- README 与页面提示一致；
- 前端测试、前端生产构建和后端测试通过。
