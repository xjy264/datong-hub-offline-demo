# 注册即用与统一权限设计

## 目标

系统不再区分超级管理员和普通用户。任何通过现有格式与密码校验完成注册的用户，都可以立即登录并使用地图、车间、站点目录和图片管理功能，无需人工审批。

## 后端行为

- 注册用户直接写入 `status = ENABLED`、`approval_status = APPROVED`、`is_super_admin = 0`。
- 登录仍校验账号存在、密码正确、账号未删除且状态为 `ENABLED`，但不再提供待审核流程。
- 所有有效登录会话统一返回 `MAP_EDIT` 权限，不再返回或判断 `USER_ADMIN`。
- 删除 `/api/admin/users` 下的用户列表、审批、禁用和管理员重置密码接口及其服务代码。
- 删除基于 `ROLE_ADMIN` 的接口分权配置；现有地图写操作继续要求登录。
- 保留数据库中的 `approval_status` 和 `is_super_admin` 字段，避免破坏既有数据与旧迁移。

## 数据迁移

新增 Flyway V9 迁移，把所有未删除用户的 `approval_status` 统一更新为 `APPROVED`，并把 `is_super_admin` 统一更新为 `0`。不修改已发布的 V1-V8。

## 前端行为

- 删除顶部“用户管理”入口。
- 删除 `/admin/users` 路由、页面组件和 `USER_ADMIN` 权限判断。
- 注册成功提示改为“注册成功，请登录”，随后返回登录页。
- 登录后的地图、车间、站点目录和上传能力保持不变。

## 安全边界

- 继续保留 HTTPS 生产门禁、Secure/HttpOnly Cookie、CSRF、防暴力请求限流、密码复杂度、手机号与姓名校验。
- “统一权限”仅取消用户角色差异，不把业务接口开放给匿名用户。
- 用户不能通过管理员接口修改他人账号，因为相关接口被删除。

## 测试与验收

- 后端测试证明注册记录为 `APPROVED`、非管理员，并且可以立即登录。
- 后端测试证明所有有效会话只返回 `MAP_EDIT`，旧管理接口不存在。
- 前端测试与生产构建通过，代码中不再存在用户管理路由和入口。
- 从现有数据库升级到 V9 后，待审核用户全部变为可登录用户。
- 真实执行“注册新手机号 → 立即登录 → 读取地图数据”。
- Docker Compose 全部服务健康。

## 发布范围

本轮默认只在独立分支实现并合并回本地 `main`，完成本地启动和验证；不自动推送 GitHub，也不部署生产。Redis 缓存分支保持不变。
