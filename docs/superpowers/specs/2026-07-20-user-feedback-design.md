# 用户操作反馈统一修复设计

## 目标

修复登录错误文案存在但不可见的问题，并统一补齐会话失效、拖动自动保存、目录操作、空名称校验及预期接口错误的用户反馈。

## 设计

1. 在应用入口全局引入 Element Plus `ElMessage` 样式，使现有成功、警告和失败消息恢复为顶部悬浮提示。
2. 401 处理区分登录页的正常会话探测与受保护页面的会话过期：前者静默，后者跳转到 `/login?reason=expired`，登录页展示一次“登录状态已过期，请重新登录。”并清除查询参数。
3. 地图按钮和区间拖动开始时保存原坐标；自动保存成功显示“位置已自动保存”，失败则恢复原坐标。接口层继续负责展示具体失败文案。
4. 新增目录和目录改名显示成功消息；改名前记录旧名称，失败后恢复，避免界面与服务器状态不一致。
5. 地图名称、地图改名和车间名称弹窗把取消与空白输入分开处理：取消静默，确认空白时显示字段级警告。
6. 接口拦截器标记已经展示或处理过的请求错误，Vue 全局错误处理器忽略这些预期错误，只把真实程序异常写入控制台。

## 边界

- 不修改后端错误文案和HTTP状态码。
- 不为登录成功、退出登录、页面切换及缩放增加冗余提示。
- 不推送远程分支，不部署生产环境。

## 验证

- Node 单元测试覆盖样式入口、401跳转、登录提示、名称规范化、位置恢复和已处理错误标记。
- 前端全量单元测试、类型检查和生产构建。
- 后端现有测试确认接口契约未受影响。
- 合并本地 `main` 后启动真实服务，通过浏览器验证错误密码提示的可见样式、空名称警告及控制台无 AxiosError。
